## Decisions locked in from prior sessions

Do not re-litigate these during execution:

1. **Caddy, not nginx**. Simpler config, automatic-TLS-is-trivial, reload-without-restart, single binary.
2. **Named tunnel, not quick tunnel**. Quick tunnels have random URLs that change on restart.
3. **Wildcard DNS** (`*.<domain>` → tunnel) instead of per-subdomain CNAMEs. Adding a project requires zero DNS changes after initial setup.
4. **Caddy's `tls internal`** with `noTLSVerify: true` in the cloudflared config, *not* Cloudflare Origin CA. Cloudflare terminates public TLS at its edge; the hop from edge to your Mac is Cloudflare-encrypted, and Caddy presents a self-signed cert that cloudflared ignores. Simpler than provisioning Origin CA certs.
5. **Caddyfile uses `import Caddyfile.d/*.conf`** pattern. One file per project in `Caddyfile.d/`, generated by `register-project.sh`, never hand-edited.
6. **Projects are separate GitHub repos** (per user preference from Phase 0). Exception: `projects/research/` which lives inside this repo because it's server-managed content (reports), not an app. For `bingo`, we'll create `github.com/Piserchia/bingo` as its own repo.
7. **`register-project.sh` is idempotent.** Running it twice with the same manifest produces the same result, no duplicate Caddy blocks or launchd plists.
8. **Healthchecks run in a script (no LLM)** per the SKILLS_REGISTRY. Phase 3 ships this script; Phase 5 adds the `server-upkeep` LLM-driven daily task that inspects healthcheck trends.
9. **Port allocation is append-only** via `projects/_ports.yml`. Register-project writes the chosen port into both `_ports.yml` and the project's `manifest.yml`; never reuses.

## Architecture refresher

```
Internet (you, anywhere)
   │
   ▼ HTTPS (Cloudflare-provided TLS)
Cloudflare edge
   │
   ▼ HTTPS (over Cloudflare Tunnel, encrypted end-to-end)
   │   noTLSVerify: true  (Caddy's self-signed cert is ignored)
cloudflared (runs on Mac Mini as launchd service)
   │
   ▼ HTTPS localhost:443  (Caddy's local CA, self-signed)
Caddy (also on Mac Mini)
   │
   ├─► yourdomain.com              → localhost:8080 (FastAPI dashboard)
   ├─► dashboard.yourdomain.com    → localhost:8080
   ├─► bingo.yourdomain.com        → file_server projects/bingo/ (static)
   ├─► market-tracker.yourdomain.com → localhost:9012 (project service)
   └─► *.yourdomain.com            → Caddyfile.d/*.conf (per-project rules)
```

### Why this specific shape

- **Cloudflare Tunnel** eliminates inbound ports from your residential ISP. No NAT traversal. No dynamic-IP issues. Survives ISP outages (Cloudflare retries).
- **Wildcard DNS** at the Cloudflare layer means adding a project doesn't touch DNS — Caddy handles routing based on the Host header.
- **Caddy with `tls internal`** means you don't need to provision real certs for the localhost hop. Caddy generates and rotates a local CA cert. cloudflared doesn't care because we set `noTLSVerify: true` on the cloudflared side.
- **Per-project snippets in `Caddyfile.d/`** means adding a project = writing one small file + reloading Caddy. Atomic, idempotent.
- **Healthchecks as a script** — because every project's healthcheck is a trivial HTTP GET against a port. No LLM needed. The interesting LLM work is *analyzing* healthcheck history, which is Phase 5's `server-upkeep`.

### New: `scripts/register-project.sh`

Idempotent registration script. Reads a project's manifest, generates the
matching Caddyfile snippet, installs a launchd plist (if service-type), and
inserts/updates the `projects` row in Postgres.

**Required behavior** (test each):
- Running with missing manifest: exits 1 with clear message.
- Running with valid static manifest: creates `Caddyfile.d/<slug>.conf`, reloads Caddy, inserts DB row.
- Running with valid service manifest: also creates `~/Library/LaunchAgents/com.assistant.project.<slug>.plist` and loads it.
- Running a second time with no changes: no-op (detect via comparing snippet content).
- Running with changed manifest (e.g., new port): updates snippet + DB row + reloads plist.
- `--dry-run` flag: prints what would change without making changes.

**Dependencies on host**:
- `yq` (YAML parser) — `brew install yq`
- `caddy` — `brew install caddy`
- `psql` (already installed)

**Script structure** (sketch — Claude Code should expand and harden):

```bash
#!/usr/bin/env bash
# scripts/register-project.sh <slug> [--dry-run]
#
# Reads projects/<slug>/manifest.yml, generates/updates:
#   - Caddyfile.d/<slug>.conf
#   - ~/Library/LaunchAgents/com.assistant.project.<slug>.plist (service-type only)
#   - projects table row (slug, subdomain, type, port, manifest_path)
# Then reloads Caddy and restarts the launchd service if present.
#
# Idempotent: running twice with no manifest changes is a no-op.
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"

DRY_RUN=false
if [[ "${2:-}" == "--dry-run" ]]; then DRY_RUN=true; fi

SLUG="${1:?usage: $0 <slug> [--dry-run]}"
MANIFEST="$PROJECT_DIR/projects/$SLUG/manifest.yml"
[[ -f "$MANIFEST" ]] || { echo "No manifest at $MANIFEST"; exit 1; }

# Parse manifest with yq
TYPE=$(yq '.type' "$MANIFEST")
SUBDOMAIN=$(yq '.subdomain' "$MANIFEST")
DOMAIN_FROM_ENV=$(grep -E '^SERVER_DOMAIN=' "$PROJECT_DIR/.env" | cut -d'=' -f2)
[[ -n "$DOMAIN_FROM_ENV" ]] || { echo "SERVER_DOMAIN not set in .env"; exit 1; }
HOSTNAME="${SUBDOMAIN}.${DOMAIN_FROM_ENV}"
PORT=$(yq '.port' "$MANIFEST")
HEALTHCHECK=$(yq '.healthcheck' "$MANIFEST")
START_CMD=$(yq '.start_command' "$MANIFEST")

SNIPPET_PATH="$PROJECT_DIR/Caddyfile.d/$SLUG.conf"
PLIST_PATH="$HOME/Library/LaunchAgents/com.assistant.project.$SLUG.plist"

# ── Generate Caddy snippet (to a temp file first so we can compare) ─────────
TMP_SNIPPET=$(mktemp)
case "$TYPE" in
  static)
    cat > "$TMP_SNIPPET" <<EOF
$HOSTNAME {
    tls internal
    root * $PROJECT_DIR/projects/$SLUG
    file_server
}
EOF
    ;;
  service|api)
    [[ "$PORT" != "null" && -n "$PORT" ]] || { echo "Port required for $TYPE"; exit 1; }
    cat > "$TMP_SNIPPET" <<EOF
$HOSTNAME {
    tls internal
    reverse_proxy localhost:$PORT
}
EOF
    ;;
  *)
    echo "Unknown type: $TYPE"; exit 1
    ;;
esac

# Idempotence: skip if unchanged
if [[ -f "$SNIPPET_PATH" ]] && diff -q "$SNIPPET_PATH" "$TMP_SNIPPET" > /dev/null; then
    SNIPPET_CHANGED=false
else
    SNIPPET_CHANGED=true
fi

if $DRY_RUN; then
    echo "Would write $SNIPPET_PATH (changed: $SNIPPET_CHANGED)"
    [[ "$TYPE" != "static" ]] && echo "Would write $PLIST_PATH"
    echo "Would reload Caddy and insert projects row"
    rm -f "$TMP_SNIPPET"
    exit 0
fi

# Commit the snippet
mkdir -p "$(dirname "$SNIPPET_PATH")"
mv "$TMP_SNIPPET" "$SNIPPET_PATH"

# ── Generate launchd plist for service-type ────────────────────────────────
if [[ "$TYPE" == "service" || "$TYPE" == "api" ]]; then
    VENV_DIR="$(cd "$PROJECT_DIR/projects/$SLUG" && pipenv --venv 2>/dev/null || echo "")"
    # (handle no-venv case: some projects might use system python or node)

    cat > "$PLIST_PATH" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key><string>com.assistant.project.$SLUG</string>
  <key>ProgramArguments</key>
  <array>
    <string>/bin/bash</string>
    <string>-lc</string>
    <string>cd "$PROJECT_DIR/projects/$SLUG" && unset ANTHROPIC_API_KEY && $START_CMD</string>
  </array>
  <key>WorkingDirectory</key><string>$PROJECT_DIR/projects/$SLUG</string>
  <key>EnvironmentVariables</key><dict>
    <key>PATH</key><string>/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin</string>
  </dict>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><dict>
    <key>SuccessfulExit</key><false/>
    <key>Crashed</key><true/>
  </dict>
  <key>StandardOutPath</key><string>$PROJECT_DIR/volumes/logs/project.$SLUG.out.log</string>
  <key>StandardErrorPath</key><string>$PROJECT_DIR/volumes/logs/project.$SLUG.err.log</string>
  <key>ThrottleInterval</key><integer>30</integer>
</dict>
</plist>
PLIST
    launchctl unload "$PLIST_PATH" 2>/dev/null || true
    launchctl load -w "$PLIST_PATH"
fi

# ── Reload Caddy ───────────────────────────────────────────────────────────
if $SNIPPET_CHANGED; then
    caddy reload --config "$PROJECT_DIR/Caddyfile" 2>&1 | tail -3
fi

# ── Upsert into projects table ─────────────────────────────────────────────
psql assistant -v ON_ERROR_STOP=1 <<SQL
INSERT INTO projects (id, slug, subdomain, type, port, manifest_path, created_at)
VALUES (
    gen_random_uuid(),
    '$SLUG',
    '$HOSTNAME',
    '$TYPE',
    $(if [[ "$PORT" == "null" || -z "$PORT" ]]; then echo NULL; else echo "$PORT"; fi),
    '$MANIFEST',
    NOW()
)
ON CONFLICT (slug) DO UPDATE SET
    subdomain = EXCLUDED.subdomain,
    type = EXCLUDED.type,
    port = EXCLUDED.port,
    manifest_path = EXCLUDED.manifest_path;
SQL

# ── Healthcheck probe ──────────────────────────────────────────────────────
if [[ "$TYPE" != "static" && "$HEALTHCHECK" != "null" ]]; then
    echo "Waiting for healthcheck..."
    for i in $(seq 1 30); do
        if curl -sf "http://localhost:$PORT$HEALTHCHECK" > /dev/null; then
            echo "✓ $SLUG healthy at https://$HOSTNAME"
            exit 0
        fi
        sleep 1
    done
    echo "⚠ $SLUG did not pass healthcheck within 30s"
    exit 2
fi

echo "✓ $SLUG registered at https://$HOSTNAME"
```

### New: `scripts/healthcheck-all.sh`

Pure bash, no LLM. Runs every 5 minutes via launchd timer. Hits every project's
configured healthcheck, updates `projects.last_healthy_at` on success, logs
failures to `volumes/logs/healthcheck.log`.

```bash
#!/usr/bin/env bash
# scripts/healthcheck-all.sh — hit every project's healthcheck, update DB.
# Designed to be fast (< 30s total for dozens of projects).
set -uo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"
LOG="$PROJECT_DIR/volumes/logs/healthcheck.log"

# Read all projects with a healthcheck set
ROWS=$(psql assistant -tAc "
  SELECT p.slug, p.port, m.healthcheck
  FROM projects p,
       LATERAL (SELECT NULL::text AS healthcheck) m  -- placeholder if not in DB
  WHERE p.port IS NOT NULL;
")

# Actually, simpler: read from manifest files directly (source of truth)
while IFS= read -r manifest; do
    SLUG=$(yq '.slug' "$manifest")
    PORT=$(yq '.port' "$manifest")
    HC=$(yq '.healthcheck' "$manifest")
    TYPE=$(yq '.type' "$manifest")
    [[ "$TYPE" == "static" ]] && continue  # no healthcheck for static projects
    [[ "$HC" == "null" || -z "$HC" ]] && continue

    URL="http://localhost:$PORT$HC"
    if curl -sf --max-time 5 "$URL" > /dev/null; then
        psql assistant -c "UPDATE projects SET last_healthy_at = NOW() WHERE slug = '$SLUG';" > /dev/null
    else
        echo "$(date -u +%FT%TZ) ✗ $SLUG $URL" >> "$LOG"
    fi
done < <(find "$PROJECT_DIR/projects" -name manifest.yml -not -path '*/.git/*')
```

Schedule it:
- `~/Library/LaunchAgents/com.assistant.healthcheck-all.plist`
- `StartInterval`: 300 (5 minutes)
- `ProgramArguments`: `/bin/bash`, `<path>/scripts/healthcheck-all.sh`

### New: `scripts/setup-tunnel.sh`

One-time tunnel setup. Runs interactively (needs browser for `cloudflared tunnel login`).

```bash
#!/usr/bin/env bash
# scripts/setup-tunnel.sh — one-time Cloudflare named tunnel setup.
# Prerequisite: domain is in your Cloudflare account with active nameservers.
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"

DOMAIN=$(grep -E '^SERVER_DOMAIN=' "$PROJECT_DIR/.env" | cut -d'=' -f2)
[[ -n "$DOMAIN" ]] || { echo "Set SERVER_DOMAIN in .env first"; exit 1; }

command -v cloudflared >/dev/null || { echo "Install: brew install cloudflare/cloudflare/cloudflared"; exit 1; }

# Login (opens browser)
if [[ ! -f "$HOME/.cloudflared/cert.pem" ]]; then
    echo "Opening browser for Cloudflare authentication..."
    cloudflared tunnel login
fi

# Create tunnel if missing
TUNNEL_NAME="ai-server"
if ! cloudflared tunnel list 2>/dev/null | grep -q "^[a-f0-9-]* *$TUNNEL_NAME "; then
    cloudflared tunnel create "$TUNNEL_NAME"
fi
TUNNEL_UUID=$(cloudflared tunnel list 2>/dev/null | awk -v name="$TUNNEL_NAME" '$2 == name { print $1 }')

# Write config
cat > "$HOME/.cloudflared/config.yml" <<EOF
tunnel: $TUNNEL_NAME
credentials-file: $HOME/.cloudflared/$TUNNEL_UUID.json

ingress:
  - hostname: "*.$DOMAIN"
    service: https://localhost:443
    originRequest:
      noTLSVerify: true
  - hostname: "$DOMAIN"
    service: https://localhost:443
    originRequest:
      noTLSVerify: true
  - service: http_status:404
EOF

# Route DNS
cloudflared tunnel route dns "$TUNNEL_NAME" "*.$DOMAIN" 2>&1 | grep -v "already exists" || true
cloudflared tunnel route dns "$TUNNEL_NAME" "$DOMAIN" 2>&1 | grep -v "already exists" || true

# Install as system service (auto-start on boot)
if ! sudo launchctl list | grep -q com.cloudflare.cloudflared; then
    sudo cloudflared service install
fi

echo "✓ Named tunnel '$TUNNEL_NAME' configured"
echo "  UUID: $TUNNEL_UUID"
echo "  Domain: $DOMAIN (wildcard + apex)"
echo "  Config: $HOME/.cloudflared/config.yml"
echo ""
echo "Test: curl -I https://$DOMAIN/health (should return 200 once Caddy is up)"
```

### New: `scripts/setup-caddy.sh`

One-time Caddy setup.

```bash
#!/usr/bin/env bash
# scripts/setup-caddy.sh — install Caddy and create the base config.
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"

command -v caddy >/dev/null || brew install caddy

DOMAIN=$(grep -E '^SERVER_DOMAIN=' "$PROJECT_DIR/.env" | cut -d'=' -f2)
[[ -n "$DOMAIN" ]] || { echo "Set SERVER_DOMAIN in .env first"; exit 1; }

mkdir -p "$PROJECT_DIR/Caddyfile.d"

cat > "$PROJECT_DIR/Caddyfile" <<EOF
{
    auto_https off
    local_certs
    admin localhost:2019
}

# Dashboard at apex and dashboard subdomain
$DOMAIN, dashboard.$DOMAIN {
    tls internal
    reverse_proxy localhost:8080
}

# Per-project snippets
import Caddyfile.d/*.conf
EOF

# Install Caddy as a launchd service if not already
if ! launchctl list | grep -q com.assistant.caddy; then
    cat > "$HOME/Library/LaunchAgents/com.assistant.caddy.plist" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key><string>com.assistant.caddy</string>
  <key>ProgramArguments</key>
  <array>
    <string>$(which caddy)</string>
    <string>run</string>
    <string>--config</string>
    <string>$PROJECT_DIR/Caddyfile</string>
  </array>
  <key>WorkingDirectory</key><string>$PROJECT_DIR</string>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><true/>
  <key>StandardOutPath</key><string>$PROJECT_DIR/volumes/logs/caddy.out.log</string>
  <key>StandardErrorPath</key><string>$PROJECT_DIR/volumes/logs/caddy.err.log</string>
</dict>
</plist>
PLIST
    launchctl load -w "$HOME/Library/LaunchAgents/com.assistant.caddy.plist"
fi

echo "✓ Caddy installed and running"
echo "  Config: $PROJECT_DIR/Caddyfile"
echo "  Per-project: $PROJECT_DIR/Caddyfile.d/"
echo ""
echo "Test: curl -k https://localhost/health"
```

- `setup-caddy.sh` — one-time: install Caddy + base config

## Dependencies

- `caddy`, `cloudflared`, `yq`, `gh` (brew-installed)
- `psql` (already there from Phase 1)

## Gotchas

- `cloudflared tunnel login` needs a browser. Run at the Mac's console.
- Routes to existing hostnames: `cloudflared tunnel route dns` is idempotent; it errors "already exists" which we grep out.
- Caddy's `tls internal` generates certs on first-request — the first hit after restart can be slow (~500ms). Subsequent are fast.
- `launchctl load -w` is needed (not just `load`) to override the disabled flag from a previous unload.
- If you edit Caddyfile.d/*.conf by hand, `caddy reload --config ./Caddyfile` picks it up without downtime.

## Rollback

If Phase 3 breaks something and you need to back out:

```bash
# Stop tunnel
sudo launchctl unload /Library/LaunchDaemons/com.cloudflare.cloudflared.plist

# Stop Caddy
launchctl unload ~/Library/LaunchAgents/com.assistant.caddy.plist

# Stop per-project services
for plist in ~/Library/LaunchAgents/com.assistant.project.*.plist; do
    launchctl unload "$plist"
done

# Stop healthcheck loop
launchctl unload ~/Library/LaunchAgents/com.assistant.healthcheck-all.plist

# The core runner/web/bot keep running — Phase 1 functionality unaffected.
```

Remove projects table rows if needed:
```bash
psql assistant -c "DELETE FROM projects;"
```

Revert to pre-Phase-3 commit:
```bash
cd "$HOME/Library/Application Support/ai-server"
git reset --hard <phase-2-commit-sha>
bash scripts/run.sh restart
```
